# 音画同步解决方案

## 问题分析

当前代码中存在的音画不同步问题：


1. **时钟同步机制简单**：仅处理视频比音频快的情况
2. **缓冲延迟未考虑**：音频缓冲区导致的实际播放延迟
3. **时间戳精度问题**：PTS/DTS转换和时钟更新不够精确
4. **帧率控制不准确**：未考虑解码和渲染延迟

## 解决方案

### 1. 改进音视频同步逻辑

#### 1.1 在native-lib.cpp中添加同步参数

```cpp
// 在全局变量区域添加
std::atomic<int64_t> g_audioBufferDuration{0};  // 音频缓冲区时长
std::atomic<int64_t> g_videoFrameDuration{0};  // 视频帧时长
std::atomic<int64_t> g_syncThreshold{40000};   // 同步阈值（40ms）
std::atomic<int64_t> g_maxFrameDelay{100000};  // 最大帧延迟（100ms）
```

#### 1.2 改进视频渲染循环

```cpp
// 在renderThread的主循环中替换原有的同步逻辑
while (true) {
    if (g_paused) {
        std::this_thread::sleep_for(std::chrono::milliseconds(100));
        continue;
    }

    AVFrame* frame = frameQueue.pop();
    if (!frame) {
        if (g_decodingFinished) break;
        continue;
    }

    // 获取当前系统时间
    int64_t current_time = av_gettime_relative();

    // 计算当前帧应该显示的时间
    int64_t frame_pts = av_rescale_q(frame->pts,
                                   fmt_ctx->streams[video_stream_index]->time_base,
                                   AV_TIME_BASE_Q);

    // 计算与音频的时间差
    int64_t audio_clock = g_audioClock.load() + g_audioBufferDuration.load();
    int64_t diff = frame_pts - audio_clock;

    // 同步决策
    if (abs(diff) < g_syncThreshold.load()) {
        // 在同步范围内，正常显示
        g_videoClock.store(frame_pts);
    } else if (diff > g_syncThreshold.load()) {
        // 视频比音频快太多，需要丢帧
        if (diff > g_maxFrameDelay.load()) {
            // 差距太大，连续丢帧
            av_frame_free(&frame);
            continue;
        } else {
            // 适度延迟显示
            std::this_thread::sleep_for(std::chrono::microseconds(diff - g_syncThreshold.load()));
        }
    } else {
        // 视频比音频慢，重复显示上一帧
        // 可以选择跳过此帧或加快显示
    }

    // 渲染帧
    renderer.renderFrame(frame);
    eglSwapBuffers(display, surface);
    av_frame_free(&frame);
}
```

// 在AudioRender.cpp中添加
int64_t AudioRenderer::getCurrentPosition() {
    if (!stream) return 0;

    int64_t framesWritten = AAudioStream_getFramesWritten(stream);
    int64_t framesRead = AAudioStream_getFramesRead(stream);
    int64_t framesPending = framesWritten - framesRead;

    // 计算缓冲区中的音频时长（微秒）
    int64_t bufferDuration = (framesPending * 1000000) / sampleRate;

    // 更新全局缓冲区时长
    g_audioBufferDuration.store(bufferDuration);

    // 返回当前播放位置
    return g_audioClock.load() - bufferDuration;
}

void AudioRenderer::setBufferSize(int32_t size) {
    if (stream) {
        AAudioStream_setBufferSizeInFrames(stream, size);
    }
}

    AVFrame *frame = av_frame_alloc();

    AVRational time_base = fmt_ctx->streams[stream_index]->time_base;

    while (true) {
        // ... 现有代码 ...

        while (avcodec_receive_frame(codec_ctx, frame) == 0) {
            // 确保时间戳有效
            if (frame->pts == AV_NOPTS_VALUE && last_pts != AV_NOPTS_VALUE) {
                // 估算时间戳
                AVRational frame_rate = av_guess_frame_rate(fmt_ctx,
                                                         fmt_ctx->streams[stream_index],
                                                         frame);
                if (frame_rate.num && frame_rate.den) {
                    frame->pts = last_pts + av_rescale_q(1,
                                                        av_inv_q(frame_rate),
                                                        time_base);
                }
            }

            if (frame->pts != AV_NOPTS_VALUE) {
                last_pts = frame->pts;
            }

### 4. 实现动态缓冲区调整

```cpp
// 在native-lib.cpp中添加缓冲区监控
void adjustAudioBuffers() {
    if (!g_audioRenderer) return;

    static int64_t lastAdjustTime = 0;
    int64_t currentTime = av_gettime_relative();

    // 每秒调整一次
    if (currentTime - lastAdjustTime < 1000000) return;

    // 获取当前缓冲区状态
    int64_t bufferDuration = g_audioBufferDuration.load();
    int32_t currentBufferSize = AAudioStream_getBufferSizeInFrames(g_audioRenderer->getStream());
    int32_t burstSize = AAudioStream_getFramesPerBurst(g_audioRenderer->getStream());

    // 动态调整缓冲区大小
    if (bufferDuration > 100000) {  // 超过100ms
        // 减小缓冲区
        int32_t newSize = std::max(burstSize * 2, currentBufferSize - burstSize);
        g_audioRenderer->setBufferSize(newSize);
    } else if (bufferDuration < 20000) {  // 少于20ms
        // 增大缓冲区
        int32_t newSize = std::min(burstSize * 8, currentBufferSize + burstSize);
        g_audioRenderer->setBufferSize(newSize);
    }

    lastAdjustTime = currentTime;
}
```

### 5. 添加音视频同步状态监控

```cpp
// 添加同步监控函数
void monitorSyncStatus() {
    static int64_t lastLogTime = 0;
    int64_t currentTime = av_gettime_relative();

    // 每5秒打印一次状态
    if (currentTime - lastLogTime < 5000000) return;

    int64_t audioClock = g_audioClock.load();
    int64_t videoClock = g_videoClock.load();
    int64_t bufferDuration = g_audioBufferDuration.load();

    int64_t diff = videoClock - audioClock;

    __android_log_print(ANDROID_LOG_INFO, "SyncMonitor",
                       "A-V diff: %lldms, Audio buffer: %lldms",
                       diff / 1000, bufferDuration / 1000);

    lastLogTime = currentTime;
}
```

## 注意事项


1. **时间精度**：使用av_gettime_relative()获取高精度时间
2. **线程安全**：确保所有全局变量的原子操作
3. **性能考虑**：避免频繁的锁竞争和内存分配
4. **边界处理**：处理PTS无效、流结束等特殊情况
5. **设备差异**：考虑不同Android设备的音频延迟差异
